# bb_task is linked here (unused -- no call site migrated yet, per the
# task-registry unification PR2b/PR3 split) solely to prove the espidf
# creation shell compiles under -Werror, mirroring the bb_power_tps546/
# bb_fan_emc2101 "prove it links" precedent below.
# bb_log_event / bb_log_http were carved out of bb_log (KB #708/#704) --
# smoke previously got the "log" bb_event topic + GET/POST /api/log/level
# routes for free via bb_log's own Kconfig-gated PRIV_REQUIRES; now that
# bb_log's base component has zero bb_http_server/bb_event deps, smoke must
# REQUIRES the two sink components explicitly to keep the same functional
# coverage (bb_sink_ws below subscribes to the "log" topic bb_log_event
# registers).
# bb_config is linked here (unused -- no call site migrated yet) solely to
# prove the component compiles under -Werror on espidf, mirroring the
# bb_task/bb_dispatch_cmd "prove it links" precedent.
# bb_config_staged (PR1b) is linked here (unused -- composition-only, no call
# site migrated yet) solely to prove the espidf creation shell compiles under
# -Werror, mirroring the bb_config/bb_task "prove it links" precedent above.
# bb_storage_nvs backs storage_typed_selftest.c's KB 791 Option A target
# proof (bb_nv <-> bb_config STR byte-identity round trip over the native
# NVS typed-entry path) -- see that file's header comment.
# bb_settings is bb's opinionated bb-config authority (KB 805/806): bb_wifi
# REQUIRES it directly and reads its wifi ssid/pass/has-creds accessors for
# the CONNECT path (the bb_wifi_creds provider seam has been collapsed).
# bb_wifi_http (PR1, KB 781) is the opt-in STA /api/wifi + /api/scan route
# bundle extracted from bb_wifi's STA core -- composed here to keep today's
# routes working; a minimal STA-only image would omit it.
# bb_wifi_ap (PR2, KB 781) is linked here (unused -- smoke does not compose
# bb_prov) solely to prove the SoftAP + captive-DNS espidf backend compiles
# and links under -Werror, mirroring the bb_task/bb_config "prove it links"
# precedent above.
# bb_storage_rtc (PR3a, WiFi-creds cluster) is linked here (unused -- an
# inert provides=storage_rtc; no consumer requires it yet, bb_nv keeps its
# own separate RTC region until PR3b) solely to prove it links and
# registers on the espidf boot path under -Werror, mirroring the
# bb_task/bb_config "prove it links" precedent above.

# DI DEMOLITION (composition migration, KB decision #735): smoke's
# composition root is now generated by `bbtool codegen` from `//
# bbtool:init` markers (see `make smoke-codegen`), mirroring
# examples/floor's `make floor-codegen` proof. The generated
# bb_app_init.c is gitignored (decision #725, build artifact, never
# committed) and absent by default -- entry_espidf.c calls
# bb_app_init_early()/bb_app_init(), so unlike floor's proof-only wiring,
# a plain `pio run -d examples/smoke` now REQUIRES `make smoke-codegen`
# (or an equivalent prior `bbtool codegen` invocation) to have run first.
set(BB_SMOKE_GENERATED_APP_INIT "${CMAKE_CURRENT_SOURCE_DIR}/generated/bb_app_init.c")
set(_smoke_srcs "entry_espidf.c" "smoke_app.c" "storage_typed_selftest.c" ${_site_srcs})
if(EXISTS "${BB_SMOKE_GENERATED_APP_INIT}")
    list(APPEND _smoke_srcs "generated/bb_app_init.c")
endif()

# codegen's resolved transitive closure (BB_AUTOWIRE_REQUIRES) can exceed
# the hand-authored SMOKE_REQUIRES above -- components whose headers
# generated/bb_app_init.c #includes but that SMOKE_REQUIRES never listed
# directly (real deps of already-required components, just not ones this
# example's SRCS previously #included itself). When the generated
# link-set fragment exists, its REQUIRES are unioned into SMOKE_REQUIRES
# (not substituted -- board-conditional appends above, e.g. bb_display*,
# bb_power_tps546, must survive) so main gets header visibility on
# everything bb_app_init.c includes.
set(BB_SMOKE_GENERATED_REQUIRES "${CMAKE_CURRENT_SOURCE_DIR}/generated/bb_autowire_components.cmake")
if(EXISTS "${BB_SMOKE_GENERATED_REQUIRES}")
    include("${BB_SMOKE_GENERATED_REQUIRES}")
    list(APPEND SMOKE_REQUIRES ${BB_AUTOWIRE_REQUIRES})
    list(REMOVE_DUPLICATES SMOKE_REQUIRES)
endif()
