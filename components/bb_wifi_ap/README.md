# bb_wifi_ap

<!-- BEGIN bbtool:brief -->
bb_wifi_ap — SoftAP + captive-DNS primitive (KB 781): pure AP bring-up and a captive DNS responder, zero HTTP. Extracted from bb_prov's former AP code; bb_prov does not call bb_wifi_ap_start()/stop(). It is a standalone primitive — callers (or the future bb_wifi_prov lifecycle FSM) invoke bb_wifi_ap_start()/stop() themselves; nothing wires it into bb_prov automatically. The AP<->STA lifecycle FSM, net-event topics, and recovery model are out of scope here — see bb_prov (soon bb_wifi_prov) for provisioning orchestration.
<!-- END bbtool:brief -->
