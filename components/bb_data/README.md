# bb_data

<!-- BEGIN bbtool:brief -->
bb_data core binding table (B1-832) -- OWNS the `key -> (desc, gather)` binding table for the future bidirectional data path (the B1-828 epic replacing bb_pub + bb_sub + all bb_sink_*). DIRECT: bb_data delegates ONLY the wire-format step to the existing bb_serialize format-dispatch registry (bb_serialize_format.h) -- it does NOT wrap bb_cache or bb_cache_serialize, and has no dependency on either.
<!-- END bbtool:brief -->
